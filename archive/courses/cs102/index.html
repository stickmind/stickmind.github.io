课程介绍

《CSE-CS101：编程方法与抽象思维——不一样的编程课》在编程方法与抽象思维上，为你提供了扎实的基础。本课程《CSE-CS102：计算机系统与编程——不一样的系统课》将在此基础上继续提升你的编程技术，在广度和深度上拓展你的编程经验。

本课程参考了斯坦福大学 CS107 课程的教学目标和大纲，从 C 语言开始，逐步深入到底层汇编语言，一步步带你梳理计算机底层的逻辑。当完全理解计算机如何执行程序并操作数据时，你将会成为一名更高效的程序员，特别是在处理调试问题、性能优化、内存管理以及程序健壮性方面。

本课程的目标是让学生掌握

  • 以复杂的内存和指针方式编写 C 程序
  • 对地址空间以及 C 程序的编译/运行时行为，能够建立一个准确的模型

获得以下能力

  • 把 C 语言转换到 x86-64 汇编语言
  • 编写适配硬件算术局限性的程序
  • 识别程序瓶颈并提高运行时性能
  • 在 Unix 环境中开发程序

贯穿整个课程，最终会带你揭开计算机基础架构的奥秘。

面向对象

学完《CSE-CS101：编程方法与抽象思维——不一样的编程课》或具备以下能力的同学：

  • 了解 C++ 基本语法

作为一名程序员，你也应该具备良好的软件工程能力，能够编写组织良好、可读性强的代码。

Unix, Command Line and C
Unix and command line tools are extremely popular tools outside of CS102 for tasks like running programs (web servers, python programs, remote programs...), accessing remote servers (Amazon Web Services, Microsoft Azure, Heroku...), programming embedded devices (Raspberry Pi, etc.), and more. Our goal for CS102 was to help you become proficient in navigating Unix. It wasn't that long ago that we were introducing the command line and commands like cd for the first time!

Our goal was also to get you up to speed writing programs in C. It wasn't that long ago that we were being introduced to our first C program!

Writing, Testing, Debugging and Profiling Code

Coding Style: Writing clean, readable code is crucial for any computer science project. Unfortunately, a fair amount of existing code is poorly-designed/documented. Our goal for CS102 was to help you write with good coding style, and read, understand and comment provided code.

Debugging (GDB): Debugging is a crucial skill for any computer scientist. Our goal for CS102 was to help you become a better debugger, specifically in the steps of narrowing in on bugs, diagnosing issues, and implementing fixes. Practically every project you work on will have debugging facilities, whether it's Python (PDB), IDEs with built-in debuggers (QT Creator, Eclipse) or web development (in-browser debugger).

Testing (Sanity Check): Testing is a crucial skill for any computer scientist. Our goal for CS102 was to help you become a better tester, especially with tasks such as writing targeted tests to validate correctness, using tests to prevent "regressions" (breaking previously-implemented functionality) and using tests to develop incrementally.

Memory Checking and Profiling (Valgrind / Callgrind): Memory checking and profiling are crucial for any computer scientist to analyze program performance and increase efficiency. Many projects you work on will have profiling and memory analysis facilities: for example, mobile development has integrated tools (Xcode Instruments, Android Profiler, etc.), and web development has in-browser tools.


You’ll see manifestations of these tools throughout projects you work on. We hope you can use your CS102 knowledge to take advantage of them!

Chars and C-Strings

Key question: How can a computer represent and manipulate more complex data like text?
Strings in C are arrays of characters ending with a null terminator! We can manipulate them using pointers and C library functions (many of which you could probably implement).

Why does this matter? Understanding this representation is key to efficient string manipulation. This is also how strings are represented in both low- and high-level languages such as C++ and Python.

Pointers, Stack and Heap

Key question: How can we effectively manage all types of memory in our programs?
Arrays let us store ordered lists of information. Pointers let us pass addresses of data instead of the data itself. We can use the stack, which cleans up memory for us, or the heap, which we must manually manage.

Why does this matter? The stack and heap allow for two ways to store data in our programs, each with their own tradeoffs, and it’s crucial to understand the nuances of managing memory in any program you write! Pointers also let us pass around references to data efficiently.
